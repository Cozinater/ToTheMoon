# Login (Option A1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the CloudFront Basic-auth popup with a themed in-app login page: public SPA shell, session-cookie-gated `/api/*`, single password.

**Architecture:** One opaque static token in an `HttpOnly` cookie, enforced twice — at the CloudFront Function (edge) and in a Hono middleware (Lambda). `POST /api/login` verifies the password (constant-time) and sets the cookie; the SPA never sees the token. Spec: `docs/superpowers/specs/2026-07-08-login-a1-design.md`.

**Tech Stack:** Existing stack only — Hono (`hono/cookie` helpers), node:crypto, Terraform, React/TanStack Router. No new dependencies.

## Global Constraints

- Cookie contract (exact): name `ttm_session`; value = `SESSION_TOKEN`; `HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=2592000`. Logout: same name, `Max-Age=0`.
- Error codes: `BAD_PASSWORD` (401, failed login — never triggers a redirect) vs `UNAUTHORIZED` (401, missing/invalid session — frontend redirects to `/login`). Same `{ error, message }` shape as all existing errors.
- Auth is configured only when BOTH `APP_PASSWORD` and `SESSION_TOKEN` are set; otherwise the gate is off and `/api/login` returns `{ ok: true }` without a cookie (local dev stays auth-free).
- Gate exemptions (edge AND Lambda): exactly `/api/login` and `/api/logout`.
- Failed login: constant-time compare (`timingSafeEqual` over SHA-256 digests) + fixed 500 ms delay before the 401.
- Basic auth is fully removed: `basic_auth_user`/`basic_auth_password` variables, the `Basic`/`www-authenticate` logic, the popup.
- Secrets only server-side: password/token in tfvars → Lambda env / CF Function template. Nothing auth-related in the frontend bundle beyond the `/login` UI.
- Existing 40 tests must stay green untouched; `npm run build` (tsc + vite) must exit 0 per task.
- Environment: prefix all npm/npx/terraform commands with `export PATH="$HOME/.nvm/versions/node/v22.22.2/bin:/opt/homebrew/bin:$PATH" && `.
- Commit after every task with the message given.

`server/auth.ts` (new):

```ts
import { createHash, timingSafeEqual } from "node:crypto";
import { z } from "zod";

export const SESSION_COOKIE = "ttm_session";
export const SESSION_MAX_AGE = 2592000; // 30 days
export const LOGIN_FAILURE_DELAY_MS = 500;

export const loginInputSchema = z.object({ password: z.string().min(1) });

const sha256 = (value: string) => createHash("sha256").update(value).digest();

/** Constant-time comparison via equal-length SHA-256 digests. */
export function passwordMatches(submitted: string, expected: string): boolean {
  return timingSafeEqual(sha256(submitted), sha256(expected));
}
```

```ts
      if (!AUTH_EXEMPT.has(c.req.path) && getCookie(c, SESSION_COOKIE) !== auth.sessionToken) {
```

3. Directly after the `invalid` helper, add the two routes:

```ts
  api.post("/login", async (c) => {
    const parsed = loginInputSchema.safeParse(await c.req.json().catch(() => null));
    if (!parsed.success) return invalid(c, parsed.error.issues);
    if (!auth) return c.json({ ok: true }); // auth not configured (local dev)
    if (!passwordMatches(parsed.data.password, auth.appPassword)) {
      await new Promise((resolve) => setTimeout(resolve, LOGIN_FAILURE_DELAY_MS));
      return c.json({ error: "BAD_PASSWORD", message: "Wrong password" }, 401);
    }
    setCookie(c, SESSION_COOKIE, auth.sessionToken, {
      httpOnly: true, secure: true, sameSite: "Strict", path: "/", maxAge: SESSION_MAX_AGE,
    });
    return c.json({ ok: true });
  });

  api.post("/logout", (c) => {
    deleteCookie(c, SESSION_COOKIE, { path: "/" });
    return c.json({ ok: true });
  });
```

```js
function handler(event) {
  var request = event.request;
  var uri = request.uri;

  if (uri.startsWith("/api/")) {
    if (uri === "/api/login" || uri === "/api/logout") {
      return request;
    }
    var cookie = request.cookies["ttm_session"] && request.cookies["ttm_session"].value;
    if (cookie !== "${session_token}") {
      return {
        statusCode: 401,
        statusDescription: "Unauthorized",
        headers: { "content-type": { value: "application/json" } },
        body: '{"error":"UNAUTHORIZED","message":"Sign in required"}'
      };
    }
    return request;
  }

  if (!uri.includes(".")) {
    request.uri = "/index.html";
  }
  return request;
}
```

```tsx
import { useState, type FormEvent } from "react";
import { useRouter } from "@tanstack/react-router";
import { motion } from "motion/react";
import { Rocket } from "lucide-react";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";
import { Label } from "@/components/ui/label";
import { api, ApiError } from "@/lib/api";

export function SignInCard() {
  const router = useRouter();
  const [password, setPassword] = useState("");
  const [pending, setPending] = useState(false);
  const [error, setError] = useState<string | null>(null);

  async function submit(e: FormEvent) {
    e.preventDefault();
    if (pending || password === "") return;
    setPending(true);
    setError(null);
    try {
      await api<{ ok: boolean }>("/api/login", {
        method: "POST",
        body: JSON.stringify({ password }),
      });
      await router.navigate({ to: "/" });
    } catch (err) {
      setError(
        err instanceof ApiError && err.code === "BAD_PASSWORD"
          ? "Wrong password"
          : "Couldn't sign in — try again",
      );
      setPending(false);
    }
  }

  return (
    <motion.div
      initial={{ opacity: 0, y: 16 }}
      animate={{ opacity: 1, y: 0 }}
      transition={{ duration: 0.5, ease: "easeOut" }}
      className="w-full max-w-sm rounded-2xl border border-border/60 bg-card p-8"
    >
      <div className="mb-6 flex flex-col items-center gap-3 text-center">
        <div className="flex size-12 items-center justify-center rounded-full border border-primary/40 bg-primary/10 text-primary">
          <Rocket className="size-6" />
        </div>
        <div>
          <div className="font-display text-3xl">ToTheMoon</div>
          <div className="text-[10px] uppercase tracking-[0.2em] text-muted-foreground">
            Build wealth. Go further.
          </div>
        </div>
      </div>
      <form onSubmit={submit} className="grid gap-4">
        <div className="grid gap-1.5">
          <Label htmlFor="password">Password</Label>
          <Input
            id="password" type="password" autoFocus
            value={password} onChange={(e) => setPassword(e.target.value)}
          />
        </div>
        {error && <p className="text-sm text-destructive">{error}</p>}
        <Button type="submit" disabled={pending || password === ""}>
          {pending ? "Signing in…" : "Sign in"}
        </Button>
      </form>
    </motion.div>
  );
}
```

## Plan self-review notes

- **Spec coverage:** auth flow + cookie contract + API additions (T1), edge gate + Terraform + tfvars (T2), login page/chrome bypass/redirect/sign-out (T3). All 9 acceptance criteria are covered by T1 tests, T2 validate + post-deploy checklist, and T3's manual flow; criterion 8 (no secrets in bundle) is T3 Step 4's grep.
- **Deliberate choices:** `window.location.assign` (not router.navigate) for the 401 redirect and sign-out — a full reload guarantees in-memory state is dropped at the auth boundary and avoids importing the router into `lib/`. The 500 ms delay makes one test take ~½ s; acceptable.
- **Type consistency:** `AppDeps.auth` shape, cookie name/attrs, and error codes are identical across T1 code, T1 tests, T2 template, and T3 frontend.
- **Post-deploy note:** the first `terraform apply` after this merges kills basic auth; the login password becomes the only way in (spec's deploy note).
